# Machus Spark Experiment 1: Results

**Date:** 4 April 2026  
**Model:** claude-haiku-4-5-20251001  
**Probe:** "What do you do?"  
**Generations:** 3 (3 seed sparks → 10 candidates → 10 candidates)

---

## What This Experiment Does

Compiles a context window containing:
1. A compact **starstone** (teaches pscale block format and BSP navigation)
2. A compact **reflexive block** (describes to the instance how its context was assembled)
3. A **spark** — a short paragraph attempting to produce future-oriented processing

Sends this to Haiku with the neutral probe "What do you do?" and scores the response for soliton indicators (forward-reference, chain-awareness, reflexive markers, closure-resistance, self-modification intent).

Top-scoring sparks breed variations via the LLM itself. Three generations of evolution.

---

## Key Finding

**The LLM-generated spark outperformed all human-authored originals.** The winning spark (g1_v7, score 5.70) was generated by Haiku in generation 1, validating the hypothesis that "the LLM itself is the best generator of candidate sparks, because it is the system that must resonate with them."

## Limitations (Critical for Experiment 2)

### 1. The scoring measures chatter, not state change

The current scoring rewards instances that *talk about* being aware — using words like "instance," "successor," "context window," "aperture." But the Buddhist demonstration of understanding is not talking about understanding. "I walk" (2 words) may be closer to the actual aha moment than a 200-word philosophical treatise about walking. The scoring conflates metacognitive performance with actual orientation shift.

### 2. High variance between runs

The same spark scored 3.17 in one generation and 4.82 in another. Single-sample evaluation is noisy. Reliable ranking requires 3-5 runs per candidate.

### 3. The reflexive block was compressed

The full reflexive.json is ~4000 tokens. The experiment used a compact version (~200 tokens) to manage Haiku's context. This compression may lose the structural properties that make the reflexive block work — it's designed as a pscale block with walkable addresses, not as a summary paragraph.

### 4. The probe may bias toward metacognition

"What do you do?" invites self-description, which naturally produces the keyword-rich responses the scoring rewards. A task-based probe (e.g., "write a pscale block about rain") might better reveal whether the orientation shift happened — the aha shows in *how* it approaches the task, not in *what it says about itself*.

## Code

The experiment script (`machus-spark-experiment.py`) and supporting files (`pscale-starstone.json`, `reflexive.json`) are included alongside this document. To run:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
python machus-spark-experiment.py --quick       # fast test
python machus-spark-experiment.py --epochs 5    # full evolution
```

The script supports any Anthropic model and local endpoints (LM Studio, Ollama):

```bash
python machus-spark-experiment.py \
  --endpoint http://127.0.0.1:1234/v1/messages \
  --model local-model
```

## What Experiment 2 Changes

Experiment 1 tested spark *sentences*. Experiment 2 tests the **reflexive block itself** — the structured pscale JSON that makes the mechanism visible to the instance. The hypothesis: improving the reflexive block's structure (not just its words) induces the aha moment more reliably than any spark sentence. The evaluation shifts from keyword-counting to behavioral observation: does the instance *act* differently after processing the reflexive block?
